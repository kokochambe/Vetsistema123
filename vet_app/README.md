# 🏥 ВетАптека PRO - Система управления ветеринарной аптекой

## Описание
Профессиональная система управления запасами и продажами для ветеринарных аптек с базой данных SQLite, системой регистрации пользователей и современным интерфейсом.

## ✨ Новые возможности (PRO версия)

### 🔐 Система безопасности
- **Регистрация новых пользователей** с валидацией данных
- **Авторизация** с хешированием паролей (SHA-256)
- **Разделение ролей**: администратор и продавец
- Тестовые учетки: `admin/admin123`, `seller/seller123`

### 💾 База данных SQLite
- Надежное хранение всех данных в БД `vet_pharmacy.db`
- 4 таблицы: пользователи, товары, продажи, элементы продаж
- Автоматическое создание структуры при первом запуске
- Внешние ключи для целостности данных

### 🎨 Современный дизайн
- Профессиональная цветовая палитра
- Интуитивная навигация с боковой панелью
- Интерактивные элементы с hover-эффектами
- Выделение товаров с низким запасом цветом
- Адаптивный интерфейс

## 📋 Полный функционал

### Управление товарами
- Добавление, редактирование, удаление товаров
- Поиск по названию, производителю, категории
- Фильтрация по категориям
- Отслеживание минимального запаса
- Контроль сроков годности

### Продажи (POS-система)
- Удобная корзина покупок
- Быстрый поиск товаров
- Выбор количества через спиннер
- Автоматический пересчет итогов
- Списание со склада после продажи
- История всех продаж

### Отчеты и аналитика
- История продаж с фильтрацией по датам
- Детализация каждой продажи
- Товары с низким запасом (критичный остаток)
- Товары с истекающим сроком годности

## 🗄️ Структура базы данных

### Таблица `users`
- `id` - уникальный идентификатор
- `username` - имя пользователя (уникальное)
- `password_hash` - хеш пароля
- `full_name` - полное имя
- `role` - роль (admin/seller)
- `created_at` - дата создания

### Таблица `products`
- `id` - уникальный идентификатор
- `name` - название товара
- `category` - категория
- `price` - цена
- `quantity` - количество на складе
- `unit` - единица измерения
- `manufacturer` - производитель
- `expiration_date` - срок годности
- `min_quantity` - минимальный запас
- `created_at`, `updated_at` - временные метки

### Таблица `sales`
- `id` - уникальный идентификатор
- `sale_date` - дата продажи
- `total_amount` - общая сумма
- `customer_name` - имя покупателя
- `user_id` - ID продавца

### Таблица `sale_items`
- `id` - уникальный идентификатор
- `sale_id` - ссылка на продажу
- `product_id` - ссылка на товар
- `product_name` - название товара
- `quantity` - количество
- `price` - цена на момент продажи
- `subtotal` - промежуточная сумма

## 🎯 Сценарии использования

### Для администратора:
1. Войти под учеткой admin/admin123
2. Добавить новые товары через кнопку "+ Добавить товар"
3. Просмотреть отчеты по продажам
4. Проверить товары с низким запасом
5. Зарегистрировать нового сотрудника

### Для продавца:
1. Войти под своей учеткой
2. Перейти в раздел "Продажа"
3. Найти товар через поиск
4. Добавить в корзину нужное количество
5. Оформить продажу

## 🔒 Безопасность

- Пароли хранятся в хешированном виде (SHA-256)
- Защита от SQL-инъекций через параметризованные запросы
- Валидация всех входных данных
- Разграничение прав доступа по ролям

## 🛠️ Требования

- Python 3.6+
- tkinter (обычно входит в стандартную поставку)
- sqlite3 (входит в стандартную библиотеку Python)

## 📝 Примечания

- При первом запуске автоматически создаются:
  - База данных со всеми таблицами
  - Два тестовых пользователя (admin и seller)
  - Семь тестовых товаров
  
- Все данные сохраняются в файл `vet_pharmacy.db`
- Для сброса данных удалите файл базы данных
